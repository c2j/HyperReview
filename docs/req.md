# HyperReview 产品需求说明书（PRD）  
—— 全球首款专为「顶级 Tech Lead 与重度代码审查者」打造的零延迟人工代码审查桌面终端  
版本：v1.0（2025 年 12 月正式定稿）  
定位语：让最后 20% 让最后 20% 的顶级 reviewer 成为神，其余人直接被卷死

## 1. 产品使命
在 AI 已接管 80% 低价值代码审查的 2025 年，  
HyperReview 要把人类仅剩的 20% 高价值审查效率再提升 5～10 倍，  
让顶级 Tech Lead 在 30 分钟内审完别人需要 3 天的 300 文件大重构，  
并甩出一份带圈选、补丁、架构热图、统计的降维打击审查报告。

## 2. 目标用户与非目标

| 是我们要做的 | 不是我们要做的 |
|--------------|----------------|
| 极致人工审查体验（零延迟、键盘驱动、离线优先） | 又一个 AI 自动审代码工具 |
| Java / SpringBoot + MyBatis + SQL/存储过程专项武器 | 通用语言全覆盖（Rust 只是加分项） |
| 支持 GitHub / GitLab / Gerrit / CodeArts / 自建 OpenAPI | 再造一个 Web 版代码仓库平台 |
| 本地审查 + 集中一次性推送评论 | 实时在线协作（激光笔语音为可选，非核心） |

## 3. 核心用户画像（付费意愿极强）
- 职称：Tech Lead / 架构师 / 技术委员会成员 / 安全审查专家  
- 日均审查量：50～300 文件  
- 技术栈：Java 后端 + MyBatis + Oracle/MySQL 存储过程 + XXL-JOB/Quartz  
- 痛点：浏览器卡顿、分散评论、重复打字、找不到历史雷区、AI 假阳性太多  
- 付费意愿：¥699～1999 / 年（他们只买最贵、最爽的）

## 5. 非功能需求（必须做到极致
| 项目 | 指标（必须达标，否则产品死亡） |
|------|--------------------------------|
| 启动到打开 10 万行 monorepo 时间 | ≤ 4 秒 |
| 切换任意历史 commit Diff 时间 | ≤ 300ms |
| 10 万行 diff 滚动、搜索、圈选 | 60fps 无卡顿 |
| 单文件审查意见输入到保存延迟 | ≤ 50ms |
| 离线状态下完整功能可用 | 100% |
| Windows / macOS / Linux 原生包大小 | ≤ 120MB |
| 数据安全 | 零云端留存，所有审查记录本地加密 |
